# FAQ

- 适用版本：`v2.0.6`
- 最后更新：`2026-02-07`

## Q1：本工具是否会上传我的本地会话内容？

默认逻辑是本地读取与本地处理。会话管理、配置写入、插件补丁均在本机执行。

## Q2：为什么“增加模型”要先备份？

因为插件 index 文件属于关键运行文件，先备份可以保证“可回滚”，避免不可逆改动。

## Q3：为什么接口检测不把 HTTP 200 直接当成功？

HTTP 200 只能说明链路层成功，不能保证业务响应有效。本工具会额外校验响应结构。

## Q4：Team 账号和普通账号的关键差异是什么？

Team 账号通常需要 `Org ID`，应用链路会写入 `OPENAI_ORG_ID`；普通账号会清理该字段。

## Q5：如果我只用 Codex CLI，不用 VS Code，哪些页面是必需的？

核心必需页面：

- 多账号切换
- Codex CLI状态
- 中转站接口（建议）
- 检查更新（建议）

## Q6：如何判断一次发版是否可发布？

满足三项即可进入发布：

1. 核心链路回归通过
2. 产物可启动且校验码已生成
3. 发布说明与版本号一致
