# 功能总览

- 适用版本：`v2.0.6`
- 最后更新：`2026-02-07`

本页按导航顺序描述每个页面的“目标、输入、输出、关键实现点”。

## 1) Codex CLI状态

- **目标**：确认本机 codex 可用性与版本状态。
- **输入**：PATH、npm 全局目录、where 查询结果。
- **输出**：本机路径/版本、最新版本、版本比较提示、一键启动入口。
- **关键点**：多来源命令发现 + 终端启动回退策略。

## 2) VSCode Codex

- **目标**：统一 VS Code 启动、插件扫描、模型规则更新。
- **输入**：VS Code 目录（可选）、工作区、插件 index 文件、模型列表。
- **输出**：插件版本信息、备份文件、规则更新结果、恢复能力。
- **关键点**：补丁链路分级（关键规则/增强规则），关键失败即中止。

## 3) config.toml配置

- **目标**：可视化编辑 `~/.codex/config.toml`。
- **输入**：从 codex 路径推断的用户目录。
- **输出**：配置读取、修改、保存、目录打开。
- **关键点**：优先按实际 codex 安装上下文推断目标路径。

## 4) opencode 配置

- **目标**：把账号池映射到 `opencode.json`。
- **输入**：选中的账号对象。
- **输出**：provider/options 字段更新、掩码展示、保存落盘。
- **关键点**：编辑区隐藏 `apiKey`，保存时恢复真实值。

## 5) 多账号切换

- **目标**：统一管理 Team/官方/中转账号并一键生效。
- **输入**：账号字段（名称、Base URL、API Key、Org ID）。
- **输出**：账号池更新、激活账号切换、配置联动写入。
- **关键点**：保存后自动选中并应用，减少“保存未生效”问题。

## 6) Codex会话管理

- **目标**：会话检索、导出、清理、继续会话。
- **输入**：`sessions/*.jsonl` 与 `history.jsonl`。
- **输出**：会话列表、详情视图、导出文件、清理统计。
- **关键点**：history 快速检索 + 深度检索回退策略。

## 7) Skill 管理

- **目标**：统一管理本地技能目录。
- **输入**：`~/.codex/skills` 下系统/用户/本地技能。
- **输出**：技能详情、导入/删除、备份、目录打开。
- **关键点**：可读取 `SKILL.md` 自动提取名称与描述。

## 8) 中转站接口

- **目标**：定位“链路/接口/模型”层面的可用性问题。
- **输入**：Base URL、API Key、模型名。
- **输出**：可用接口列表、模型可用性结论、诊断明细。
- **关键点**：成功判定基于状态码 + 响应结构，不是仅看 200。

## 9) OpenAI官网状态

- **目标**：展示 OpenAI 官方状态组件健康度。
- **输入**：状态 API summary 数据。
- **输出**：总体状态 + 组件分组（异常高亮）。

## 10) 检查更新

- **目标**：及时发现版本更新并可视化提醒。
- **输入**：GitHub Releases API 数据。
- **输出**：当前版本、最新版本、变更摘要、红点数量。
- **关键点**：自动检查 + 手动检查并存。
