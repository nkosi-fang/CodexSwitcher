# 功能总览

本页按左侧导航顺序描述每个页面的“目标、输入、输出、实现关键点”。

## 1) Codex CLI状态

**目标**：确认本机 codex 可用性，并给出升级路径。  
**输入**：本机环境（PATH、npm 全局目录、where 查询）。  
**输出**：本机路径、本机版本、最新版本、版本差异提示、一键启动入口。  
**关键点**：多来源查找 codex，可回退到不同终端拉起方式。

## 2) VSCode Codex

**目标**：把 VS Code 启动、插件扫描、模型规则更新集中在一个页面。  
**输入**：VS Code 安装目录（可选）、工作区、插件 index 文件、模型列表。  
**输出**：

- 启动 VS Code / 修复 WebView
- 扫描并显示插件版本与最新版本信息
- 备份并更新模型规则，支持恢复最近备份

**关键点**：模型补丁分“关键规则/增强规则”，关键失败会中止，增强失败给提示。

## 3) config.toml配置

**目标**：可视化编辑 `~/.codex/config.toml`。  
**输入**：推断出的 codex 用户目录。  
**输出**：配置内容读取、保存、打开目录。  
**关键点**：优先依据 codex 路径推断用户目录，避免写错用户环境。

## 4) opencode 配置

**目标**：把账号池配置映射到 `opencode.json`。  
**输入**：账号池中选定账号。  
**输出**：`provider/options` 区域自动更新，并可手工编辑保存。  
**关键点**：编辑区对 `apiKey` 做掩码显示，保存时恢复真实值，降低误泄露风险。

## 5) 多账号切换

**目标**：统一管理 Team/官方/中转账号，并一键生效。  
**输入**：账号基础字段（名称、Base URL、API Key、Org ID）。  
**输出**：账号池更新 + 当前激活账号切换 + 配置文件同步写入。  
**关键点**：保存后自动选中并应用，减少“保存了但没生效”的操作偏差。

## 6) Codex会话管理

**目标**：对本地会话做检索、导出、清理和继续会话。  
**输入**：`sessions/*.jsonl` 与 `history.jsonl`。  
**输出**：会话列表、详情渲染、导出文件、清理结果。  
**关键点**：先走 history 快速检索，再回退深度检索；右键可继续到 CLI/VS Code。

## 7) Skill 管理

**目标**：统一管理本地技能目录。  
**输入**：`~/.codex/skills` 下系统/用户/本地技能。  
**输出**：技能列表、详情、导入/删除、备份与目录打开。  
**关键点**：支持读取 `SKILL.md` 自动提取名称与描述。

## 8) 中转站接口

**目标**：定位“账号问题、链路问题、接口问题、模型问题”。  
**输入**：Base URL、API Key、模型名。  
**输出**：可用接口列表、模型可用性结论、详细诊断摘要。  
**关键点**：成功判定基于“状态码 + 响应结构”，不是只看 HTTP 200。

## 9) OpenAI官网状态

**目标**：聚合官方状态页组件信息。  
**输入**：`status.openai.com` 的 summary API。  
**输出**：总体状态 + 组件分组展示（异常高亮）。

## 10) 检查更新

**目标**：发现新版本并在导航给出红点数量提醒。  
**输入**：GitHub Releases API。  
**输出**：当前版本、最新版本、更新内容摘要、版本差异计数。  
**关键点**：支持自动周期检查与手动检查共存。
